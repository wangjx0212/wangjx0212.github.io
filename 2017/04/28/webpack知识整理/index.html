webpack知识整理

发表于 2017-04-28

前言

webpack是一款web构建模块化加载器兼打包工具，使用JS作为载体将把有依赖关系的各种文件打包成一系列静态资源，作为模块来使用和处理。

简单点来说，webpack就是一个配置文件，把不同文件定义为模块，处理的方式依赖不同的工具，经过相同的编译工序loader==>plugin。

Webapck整体架构 (以webpack.config主要部分进行划分)

1.entry :定义整个编译过程的起点，页面中的入口文件；
2.output : 定义整个过程的终点，指页面通过webpack打包生成后的目标文件和路径；
3.module ：定义模块module的处理方式，也是最关键的配置，告知webpack不同的文件需要什么样的处理；
4.plugin : 对编译完成后的内容进行二度加工，也就是需要的插件。
5.resolve ：定义了解模块路径时的配置和别名，可选填写。

重点记录大名鼎鼎的loaders

通过使用不同的loader，webpack通过调用外部的脚本或者工具，可以对各种各样的的格式的文件进行处理，

Loaders需要单独安装并且需要在webpack.config.js下的modules关键字下进行配置，
其包括选项如下，

  • test ： 匹配loaders所处理的文件的扩展名的正则表达式(必须)
  • loader：loader的名称(必须)
  • include/exclude :手动添加必须处理的文件(文件夹)或屏蔽不需要的文件(文件夹)(可选);
  • query: 为loaders提供额外的设置选项(可选)
