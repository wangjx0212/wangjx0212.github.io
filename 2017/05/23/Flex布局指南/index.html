Flex布局指南

发表于 2017-05-23

前言

传统的WEB布局解决方案，是基于盒装模型、依赖于display属性+position属性+float属性。针对于一些特殊的布局就不太方便，如垂直居中、响应式等；

Flexbox是一个模块，而不是单一的属性，它分为容器上的属性（父元素，也称为‘flex容器’），而有一些是容器子元素上的属性（常称为‘flex项目’）。

Flex布局的主要思想：容器默认存在两个轴，水平的主轴（main axis）和垂直的交叉交叉轴。主轴的开始位置（与边框的交叉点）叫做main start ，结束位置叫做 main end；
交叉轴的开始位置叫做 cross start ，结束位置叫做 crose end。
项目默认沿着主轴排列。 单个项目占据的主轴空间叫做 main size，占据的交叉轴空间叫做 cross size 。

display

定义一个Flex 容器，根据其取得值来决定是内联还是块。Flex容器会为其内容建立新的伸缩格式化上下文。

注意: 设为Flex布局后，子元素的float、clear和vartical-align属性将失效。

容器的属性

以下6个属性设置在容器上

1. flex-direction
2. flex-wrap
3. flex-flow
4. justify-content
5. align-items
6. align-content

flex-direction

这个是用来创建方轴，从而定义Flex项目在Flex容器中放置的方向，Flexbox是一种单方向的布局概念。

它可能有四个值

1.row(默认值) ：主轴为水平方向，起点在左端
2.row-reverse ：主轴为水平方向，起点在右端
3.column ：主轴为垂直方向，起点在上沿
4.column-reverse ：主轴为垂直方向，起点在下沿

flex-wrap属性

默认情况下，Flex项目都尽可能在一行显示。这就可以根据flex-wrap的属性值来进行改变，让Flex项目多行显示。

它可能有三个值

1.nowrap(默认值) ：单行显示
2.wrap ：多行显示，第一行在上方
3.wrap-reverse ：多行显示，第一行在下方

flex-flow

flex-flow属性是flex-direction和flex-wrap属性的简写 默认值为 row nowarp。

justify-content

justify-content 属性定义了项目在主轴上（main-Axis）的对齐方式。

它可能取5个值，具体对齐方式与轴的方向有关。假设主轴为从左到右。

1.flex-start(默认值) ：左对齐（项目一行的起始位置对齐）
2.flex-end ：右对齐（项目一行的结束位置靠齐）
3.center ：居中
4.space-between ：两端对齐项目之间的间隔相等
5.space-around ：每个项目两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍

align-items

align-items 属性定义项目在交叉轴上如何对齐，类似于justify-content，但是是另一个方向（Cross-Axis）

它可能取五个值，具体对齐方式与轴的方向有关。假设交叉轴为从上到下

1.flex-start ：交叉轴的起点对齐
2.flex-end ：交叉轴的终点对齐
3.center ：交叉轴的中点对齐
4.stretch(默认值) ：如果项目未设置高度或者auto，将占满整个容器的高度
5.baseline ：项目的第一行文字的基线

align-content属性

align-content 属性定义了多根轴线的对齐方式。 前面讨论的warp属性，我们在Flex容器中添加了更多的Flex容器，让容器中的项目多行排列，如果项目只有一行，该属性
不起作用。

该属性可能取6个值

1.flex-start ：与交叉轴的起点对齐
2.flex-end ：与交叉轴的终点对齐
3.center ：与交叉轴的中点对齐
4.stretch(默认值) : 轴线占满整个容器的高度
5.space-around ：每个轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍
6.space-between ：与交叉轴两端对齐，轴线之间的间隔平均分布

项目的属性

以下6个属性设置在项目上

1.order ：
2.flex-grow ：
3.flex-shrink ：
4.flex-basis :
5.flex ：
6.align-self ：

order

默认情况下，Flex项目是按文档源的流顺序排列，然而可在Flex容器中通过order属性来控制Flex项目的顺序源。

根据order重新排序伸缩项目，有最小（负值最大）order的伸缩项目拍第一，若有相同的order值，这些项目按照文件顺序排。

默认情况下，所有Flex项目的order值都是0.

flex-grow

flex-grow 属性定义项目的放大比例。默认为0，即如果存在剩余空间，也不放大。

如果所有项目的flex-grow设置为1，表示Flex容器中的Flex项目具有哦相等的尺寸，如果一个项目的flex-grow属性为2，其他项目为1，则前者占据的剩余空间将
比其他项目多一倍（其他Flex项目的flex-grow值为1）。
注意: flex-grow 取负值将失效

flex-shrink

flex-shrink 定义Flex项目的缩小比例,默认为1，即如果空间不足，该项目将缩小。

注意: flex-shrink 取负值将失效

flex-basis属性

flex-basis属性定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是够有多余空间。它的默认值auto，即项目的本来大小。

他可以设为跟width或height属性一样的值（比如350px）,则项目将受到限制，占据固定空间。
注意： 如果flex-basis属性的值为0时，也需要使用单位。

flex 属性

flex 属性时 flex-grow，flex-shrink和flex-basis的简写，默认值为 0 1 auto。后两个属性可选。

该属性由两个快捷值，auto（1 1 auto）和 none （ 0 0 auto ）
建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值。

align-self 属性

align-self 属性允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items
属性，如果没有父元素，则等同于stretch。它可以更进一步让我们更好地控制弹性项目。

该属性可能取6个值，除了auto ，其他都与 align-items属性完全一样。

Auto-margin 对齐

当Flex项目上使用margin：auto时，值为auto的方向（左、右或者两者都是）会占据所有剩余空间。
当一个Flex项目使用自动外边距时，justify-content属性就不起作用了。
