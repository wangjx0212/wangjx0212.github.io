一个基于Vue+Node的前后端分离案例

发表于 2017-05-26

前言

前端开发过程当中一直是基于后端的环境，而后端环境搭建是一项学习成本蛮大的技能，最近入坑了vue，看了不少的SPA案例，前后端分离的概念也了解些，因此决定
体验把前后端分离的全栈式开发

简介

一个前后端分离的案例，前端 Vue.js ,后端 express，数据库mongodb

开发环境

需要本地安装 node、npm（cnpm）、vue-cli、mongodb

初始化

首先vue-cli 进行初始化项目

默认8080端口，出现vuejs欢迎的界面表示成功
接着把本地mongodb服务跑起来，具体参考上一篇blog

后端开发

先搭服务环境，使用npm安装 express、mongodb等必须要的依赖

express + mongodb + mongoose + body-parser + nodemon + cookie-parser + express.static

body-parser: express中间件，用于解析客户端请求的body中的内容，内部使用JSON编码处理，url编码处理以及对于文件的上传处理。

bodyParser导入时应写在路由前面。

然后再项目跟目录下创建app.js，编写好启动express服务器的代码

使用 nodemon ，进行热加载开发，cookie-parser 中间件可以操作cookie；

PS：个人局部安装成功，但是不知道为什么用不了，所我选择全局安装了
浏览器访问 localhost:3000 ，出现 res.send()中的内容，就代表成功了。
然后添加需要的数据，根目录下先创建一个server目录，代表服务端存放的文件，接着在server下再分别新建 models和router目录；
models目录存放数据模型，mongoose的每个数据model需要一个schema生成，新建Account.js文件或其他，也可以再细分开来，用来提供基础数据，

定义了几个基本信息，最后将model导出。

Mongoose是在node异步环境下对mongodb进行敏捷操作的对象模型工具,具体参考如下：
Mongoose介绍和入门

前端开发

element ul + axios + vue-router + vuex

首先选择安装依赖，

把一些不要的文件都删除，然后再src/components下新建两个三个组件login.vue,CreateAccont.vue和index.vue；
然后再src/router建立前端路由文件,这三个路由之间进行切换，然后把它放在App.vue里。
前端路由

前后端的数据请求，因为express是在3000端口启动的,而前端则在8080端口,由于跨域所以要配置好vue-cli的proxyTable 选项,位于config/index.js,改写proxyTable.

这个时候前端用axios访问 ‘/api’ 的时候,就会被代理到 ‘http://localhost:3000/api',从而获得需要的数据.
在前后端数据请求的环境搭好之后就是页面的编写了，参考element-ui组件库，可自由搭页面，关键的是访问后端对数据的增删盖查，
这些的操作呢，都写在methods对象里面，写好主要的方法后 ，可click在对应的组件按钮上。

结语

前端开发完成后,就能用webpack打包了,注意将config/index.js文件里面的productionSourceMap设为false,
不然打包出来文件很大,最后用express.static中间件将webpack打包好的项目目录’dist’作为express静态文件服务的目录.
