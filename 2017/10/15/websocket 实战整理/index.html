websocket 实战整理

发表于 2017-10-15

前言

相较于http这种单向的，只能由客户端发起请求，且不能持久链接的协议，websocket可以做到客户端实时获得服务器端推送的消息。

特点：

  • 建立在TCP协议之上，服务器端比较容易实现；
  • 和HTTP协议有良好的兼容性，默认端口80，握手阶段采用HTTP协议，能通过HTTP代理服务器；
  • 数据格式轻量，性能开销小，通信高效；
  • 可以发送文本，二进制等；
  • 没有同源限制，客户端可以和任意服务器通信；
  • 协议标识符是ws（如果加密，则为wss），服务器网址为URL；

从7月中旬到现在，一直忙于负责一个基于websocket的类客服系统，vue+node的开发技术栈，接下来的博客会记录这个系统开发过程中所用到的东西。过来是这个项目已经开始，被告知Node的socket.io模块不可用，所以选择了ws这个模块。

客户端的API：

WebSocket对象作为一个构造函数，用于新建WebSocket实例：

webSocket.readyState

readyState属性返回实例对象的当前状态，共4种：

  • CONNECTING:值为0，表示正在连接；
  • OPEN：1，表示连接成功，可以通信；
  • CLOSING：值为2，表示连接正在关闭；
  • CLOSED：值为3，表示连接已经关闭，或者打开你连接失败；

webSocket.onopen

实例对象的onopen属性，用于指定连接成功后的回调函数

webSocket.onclose

实例对象的onclose属性，用于指定连接关闭后的回调函数。

webSocket.onmessage

实例对象的onmessage属性，用于指定收到服务器数据后的回调函数。

服务器数据可以发送的是文本，也可以是二进制数据(blob对象或者Arraybuffer对象)

除了动态判断还可以使用binaryType 属性，显示指定收到的二进制数据类型。

webSocket 客户端心跳机制

在websocket中遇到客户端网络关闭的情况，服务器不会触发onclose事件，这样会：

  1. 多余的连接；
  2. 服务器继续会向客户端发送数据，数据丢失。
    所以需要设计一个心跳机制来检测客户端和服务器的连接状态。

要解决的问题：

  1. 连接之后，每10s发一个心跳，服务器同样返回一个心跳，用来表示服务器没挂；
  2. 断线重连，断开网络后，心跳包无法发送，这时如果当前时间距离上次成功心跳超过20s，说明连接出现问题，需要关闭连接；
  3. 第一次关闭websocket会尝试重连，设置一个时间限制，10s，如果10s内能连接上(接上网络)就可以继续发消息，如连接不上就关闭不会重连；
  4. 30s内收不到服务器的消息，个人认为服务器挂了，调用close事件，进入item3。

需要准备

  1. 一个定时器ws.keepAliveTimer,用来没10秒发一个心跳
  2. 上次心跳成功的时间，ws.last_health_time以及当前时间 let time = new Date().getTime();
  3. 断开连接时的时间，reconnect ,因为close事件发生后需要重连10s;
  4. 断开连接时需要保存ws对象tempWs,
  5. 一个定时时间为30s的setTimeOut 定时器ws.receiveMessageTimer
