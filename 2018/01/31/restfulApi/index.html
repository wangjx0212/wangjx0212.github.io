RESTFul Api

发表于 2018-01-31

前言

REST 即Representational State Transfer的缩写，直译为“表现层状态转化”，通俗讲，改变url的写法，url里只带上名词，让 它带有状态、语义化。

REST风格的API介绍。

REST架构使用CRUD原则，对于资源只需要4种行为：创建(Create)、获取(Read)、更新(Update)和销毁(Delete),与之对应的是HTTP协议的4种请求方法状态：
POST、GET、PUT(PATCH)和DELETE

例子 ：

  • GET /zoos: 列出所有动物园
  • GET /zoos/ID:获取某个指定动物园的信息
  • POST /zoos: 新建一个动物园
  • PUT /zoos/ID: 更新某个指定动物园的信息（提供该动物园的全部消息）
  • PATCH /zoos/ID:更新某个指定动物园的消息(提供该动物园的部分消息)
  • DELETE /zoos/ID:删除某个动物园

定义返回的数据格式

API返回来的数据格式，选择更轻量级的JSON，在返回的数据中需要知道当前的请求是否成功，如果成功返回详细的信息，如果失败，返回相应的错误信息

根据code来判断问题，code:0,表示请求成功。

扩展response对象

给response对象扩展两个方法，apiSuccess()和apiError(),分别用于响应请求成功和请求失败时的结果。
